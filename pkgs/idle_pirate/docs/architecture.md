# Idle Pirate - Architecture

This document describes the architecture and project structure for the Idle Pirate game.

## Project Structure

We follow a simple layered approach:

```text
pkgs/idle_pirate/
├── lib/
│   ├── main.dart             # Application entry point and persistence init
│   ├── models/               # Data models (Game State, Item)
│   │   ├── game_state.dart
│   │   └── item.dart         # Combined model for equipment, crew, and fleet
│   ├── state/                # State management logic
│   │   └── game_controller.dart
│   └── ui/                   # UI Widgets and Screens
│       └── screens/
│           └── game_screen.dart # Main screen with extracted widgets (ItemTile, ItemGroup)
└── docs/                     # Documentation (this folder)
```

## State Management

We use a simple and reactive state management system.

- **ChangeNotifier**: We use Flutter's built-in `ChangeNotifier` to hold the game state and notify listeners when data changes.
- **GameController**: A central controller class extending `ChangeNotifier` handles the game loop (ticking periodically to generate passive income) and actions (like clicking or buying upgrades).

## Persistence

Idle games require progress to be saved automatically.

- We use the **Hive** package for persistence. Hive is a lightweight and blazing fast key-value database written in pure Dart, which is ideal for a mobile and web game.
- It allows us to store complex objects and lists easily.

## Game Loop

The game loop is critical for an idle game.
- We implement a timer that ticks periodically for UI updates and income calculation.
- On each tick, the passive income generated by the crew and fleet is calculated and added to the total gold.
