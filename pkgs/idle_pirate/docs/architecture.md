# Idle Pirate - Architecture

This document describes the proposed architecture and project structure for the Idle Pirate game.

## Project Structure

We will follow a simple, feature-oriented structure as the project grows, but for the initial phase, a simple layered approach will suffice:

```text
pkgs/idle_pirate/
├── lib/
│   ├── main.dart             # Application entry point
│   ├── models/               # Data models (Game State, Crew, Upgrades)
│   │   ├── game_state.dart
│   │   ├── crew_member.dart
│   │   └── upgrade.dart
│   ├── state/                # State management logic
│   │   └── game_controller.dart
│   └── ui/                   # UI Widgets and Screens
│       ├── screens/
│       │   └── game_screen.dart
│       └── widgets/
│           ├── chest_widget.dart
│           └── crew_list_widget.dart
└── docs/                     # Documentation (this folder)
```

## State Management

For a small to medium-sized idle game, a simple and reactive state management system is ideal.

### Proposed Approach
- **ChangeNotifier**: We will use Flutter's built-in `ChangeNotifier` to hold the game state and notify listeners when data changes.
- **GameController**: A central controller class extending `ChangeNotifier` will handle the game loop (ticking every second to generate passive income) and actions (like clicking or buying upgrades).

### Future Considerations
If the game complexity increases significantly, we might consider migrating to a more robust state management solution like **Riverpod** or **Bloc**.

## Persistence

Idle games require progress to be saved automatically.

- We will use the **Hive** package for persistence. Hive is a lightweight and blazing fast key-value database written in pure Dart, which is ideal for a mobile and web game.
- It allows us to store complex objects and lists easily, which will be useful as the game state grows.

## Game Loop

The game loop is critical for an idle game.
- We will implement a timer that ticks periodically (e.g., every 100 milliseconds for smooth UI updates or every 1 second for simpler logic).
- On each tick, the passive income generated by the crew will be calculated and added to the total gold.
